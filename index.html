Hi-Fidelity Prototypes

Dashboard prototype is completely based on Fiori Overview page design. The overview page (OVP) is a new data-driven Fiori app that provides all the information a user needs in a single page, based on the user’s specific domain or role. It allows the user to focus on the most important tasks, enabling faster decision-making as well as immediate action.
The overview page employs cards (containers of content) based on smart template technology as a UI framework for organizing large amounts of information on an equal plane within the page. Information can be visualized in an attractive and efficient way using different types of cards.
In short, the overview page is a simple, one-stop-shop, data-driven Fiori app that gives the user all essential information. It allows the user to view, filter, and react easily from anywhere.

Access Analysis application is biuld based on Fiori object page floorplan is the recommended way to represent objects in SAP Fiori. The object page, provides a flexible header, alternative anchor or tab navigation, and a flexible responsive layout, make it adaptable for a wide range of use cases.

Wireframing

After refining the way the UI should look, I make higher fidelity wireframes. These typically reflect the incorporated refinements based on pre-build tests done on earlier prototypes, and the observations and results gathered during those tests.

I go with a mobile-first approach, designing and wireframing first for a small screen, and then working my way up to larger screens sizes as I proceed towards my goal of a responsive design. I follow this path because it gives me a better way of organizing content on each screen type since I get more space for each increasing device screen size.

Here I showcase sample mobile wireframes that I've created using Balsamiq, and also some sample web wireframes to complement the goal of responsive design.






High Fidelity Mockups

Once the structure of the content on the user interface has been defined and refined, I proceed to create high-fidelity mockups of the same, incorporating color and other styles that would make the prototype look a static version of the almost-finished product.
Here again, I go with a mobile-first approach and work my way up towards web.
